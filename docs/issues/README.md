# Issues

This directory is the local issue tracker for Tradesites AI Sales Trainer until the repo has a remote GitHub project.

## Workflow

- New work starts in `BOARD.md`.
- Each issue should be a vertical slice: data, API, UI, tests, and docs where relevant.
- Mark issues as `AFK` when an agent can implement them without new decisions.
- Mark issues as `HITL` when they need human review or a product decision.
- Keep local user data out of issues. Link to fixture names, not private transcripts.

## Labels

- `ready-for-agent`: fully specified and implementation-ready.
- `needs-triage`: needs scope or acceptance criteria.
- `needs-info`: waiting on a human answer.
- `ready-for-human`: needs human judgment or manual testing.
- `wontfix`: intentionally not planned.
